<context>
This is an enhancement to the existing `plugin-creator` plugin. No new plugin — everything
goes into `plugins/plugin-creator/`. The `rwr:doc-to-skill` agent handles converting the
fetched human-facing docs into AI-facing reference files. The fetch script is `.cjs` because
Node.js is the only guaranteed runtime across all platforms Claude Code runs on.

Key constraint re-learned: `hook-creator` skill explicitly prohibits bash and Python for hook
scripts in this repo. The reason is cross-platform compatibility — Windows users would be broken
by bash scripts.

The GitHub Copilot hooks use a completely different schema from Claude Code:
- File: `.github/hooks/` on default branch, `version: 1`
- Event names: camelCase (`sessionStart`, `preToolUse`, `userPromptSubmitted`, `postToolUse`, `errorOccurred`)
- Commands: dual `bash`/`powershell` keys per hook entry
- No `matcher` concept
- `cwd` and `env` fields per hook entry
</context>
