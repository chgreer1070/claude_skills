# Research Integration Opportunities System

Automated cross-referencing system that analyzes research files and identifies integration opportunities with existing repository capabilities.

## Overview

The Research Context Agent processes markdown files in `research/*/` and appends structured **Integration Opportunities** sections that identify:

1. **Enhances Existing** - How research could improve current skills/agents/hooks
2. **New Skill Candidates** - Workflows warranting standalone skills
3. **New MCP Server Candidates** - APIs/data sources for Claude Code integration
4. **Cross-References** - Related research files and synergies

## Components

### 1. Research Context Agent

**Location**: `.claude/agents/research-context-agent.md`

Specialized agent that:
- Reads and understands research content
- Searches the repository for connections across 5 dimensions
- Appends structured integration opportunities sections
- Follows quality gates (concrete over vague, skip empty sections)

### 2. Command Interface

**Location**: `.claude/commands/process-research-integration.md`

Claude Code command: `/process-research-integration`

Options:
- `--category <name>` - Process single category
- `--file <path>` - Process single file
- `--force` - Reprocess existing sections
- `--dry-run` - Preview what would be processed

### 3. Batch Processing Scripts

**Interactive Script**: `batch-process-research.sh`
- Guides through processing each file
- Shows Task tool prompts to copy/paste
- Tracks progress with file counts
- Checks for existing sections

**Python Script**: `process-research-integration.py`
- Analysis and reporting tool
- Counts existing integration opportunities
- Can be extended for automation

## Usage

### Process a Single File

Use the Task tool with this prompt:

```
You are the research-context-agent as defined in /path/to/.claude/agents/research-context-agent.md

Please process this research file:
/path/to/research/category/filename.md

Follow the three-phase process:
1. Absorb: Read and understand the research file
2. Search & Match: Find connections across the 5 dimensions
3. Append: Add or REPLACE the Integration Opportunities section

Use the structured format with tables and lists as specified.
Remember: Concrete over vague, skip empty sections, no false positives.
```

### Batch Process Multiple Files

```bash
# Interactive mode - guides through each file
./batch-process-research.sh

# Process specific category
./batch-process-research.sh --category developer-tools

# Dry run - see what would be processed
./batch-process-research.sh --dry-run

# Process single file
./batch-process-research.sh --file research/developer-tools/loguru.md
```

### Using the Command

In Claude Code:

```
/process-research-integration
/process-research-integration --category mcp-ecosystem
/process-research-integration --file research/developer-tools/jscpd.md
```

## Output Format

```markdown
---

## Integration Opportunities

> Auto-generated by research-context-agent. Review before acting.

### Enhances Existing

| Target | Type | How |
|--------|------|-----|
| `plugins/X/skills/Y/` | skill | Concrete 1-2 sentence description |
| `.claude/agents/Z.md` | agent | Concrete 1-2 sentence description |

### New Skill Candidates

- **Proposed skill name**: What Claude would learn to do, referencing specific research capabilities

### New MCP Server Candidates

- **Proposed MCP name**: What tools/resources it would expose

### Cross-References

- Related research: `research/category/other-file.md` — Why they're related
```

## Quality Standards

The agent enforces these rules:

✅ **Concrete over vague** - "Could enhance holistic-linting by adding jscpd duplicate detection" NOT "Could be useful for code quality"

✅ **Skip empty sections** - If no matches, omit that subsection entirely

✅ **No false positives** - Only genuine, specific connections

✅ **Preserve existing content** - NEVER modify above the `---` separator

✅ **Idempotent** - Can re-run without duplication

## Development

### Adding New Search Dimensions

Edit `.claude/agents/research-context-agent.md` to add new dimensions to Phase 2: Search & Match.

### Customizing Output Format

Modify the Phase 3: Append section in the agent definition.

### Extending the Scripts

The Python script (`process-research-integration.py`) can be extended with:
- Automatic file iteration
- Parallel processing
- Integration with CI/CD
- Output validation

## Files Modified

This system only modifies research files in `research/*/`. It:
- Preserves all content above the Integration Opportunities section
- Replaces existing Integration Opportunities sections
- Appends new sections if none exist
- Never modifies README.md files

## Troubleshooting

**Issue**: Agent generates vague suggestions
**Solution**: Edit the prompt to emphasize "concrete over vague" rule

**Issue**: Too many false positives
**Solution**: Reduce search breadth, increase specificity requirements

**Issue**: Missing obvious connections
**Solution**: Add more search keywords, check grep patterns

**Issue**: Format doesn't match
**Solution**: Verify agent is using latest definition from `.claude/agents/research-context-agent.md`
