### How This Currently Works: Token Resolution in fetch_gitlab_mr.py

`fetch_gitlab_mr.py` is a PEP 723 inline-script at `.claude/skills/create-merge-request-changelog/scripts/fetch_gitlab_mr.py`. It is executed via `uv run` directly (no package install required) and uses the header at lines 2-5 to declare its own dependencies: `typer>=0.21.0`, `python-gitlab>=4.0.0`, and `gitpython>=3.1.0`.

The token resolution flow starts in `_resolve_token()` (lines 91-115). The function first attempts to read `GITLAB_TOKEN` from the environment via a walrus-operator expression at line 102. If that is truthy, it returns the token immediately. If both `GITLAB_TOKEN` and `GITLAB_PRIVATE_TOKEN` are absent, execution falls through to the subprocess block at lines 105-115. That block calls `shutil.which("glab")` to locate the glab binary on `PATH`. If glab is not found, the function returns `None`. If glab is present, it runs `subprocess.run([glab_path, "config", "get", "token", "-h", _get_gitlab_host()], capture_output=True, text=True, check=True)` and returns the stripped stdout. A `subprocess.CalledProcessError` (non-zero exit) causes a `None` return.

The single caller of `_resolve_token()` is `get_gitlab_client()` at line 118. It raises `GitLabFetchError` with the message `"No GitLab token found. Set GITLAB_TOKEN or configure glab."` when the token is `None`. After obtaining the token it constructs `gitlab.Gitlab(f"https://{_get_gitlab_host()}", private_token=token)`.

The `_get_gitlab_host()` function (line 73) delegates to `_parse_git_origin()` (line 56), which is decorated with `@lru_cache(maxsize=1)` and reads the git remote origin URL exactly once per process invocation. This function is currently called inside the subprocess block of `_resolve_token()` to build the `-h` argument. After the deletion, `_get_gitlab_host()` will no longer be called from `_resolve_token()` — it remains called from `get_gitlab_client()` at line 132 to construct the Gitlab URL, and from `_get_project_path()` indirectly, so it is not deleted.

The imports `import shutil` (line 17) and `import subprocess` (line 18) are used exclusively by the subprocess block. After the block is removed, both become unused. def _resolve_token() -> str | None:
    """Resolve GitLab API token from environment variables.

    Checks in order:
    1. GITLAB_TOKEN environment variable
    2. GITLAB_PRIVATE_TOKEN environment variable

    Returns:
        Token string, or None if no token found.

```python
def get_gitlab_client() -> gitlab.Gitlab:
    """Get authenticated GitLab client."""
    token = _resolve_token()
    if not token:
        msg = "No GitLab token found. Set GITLAB_TOKEN or GITLAB_PRIVATE_TOKEN env var."
        raise GitLabFetchError(msg)
    return gitlab.Gitlab(f"https://{_get_gitlab_host()}", private_token=token)
```
