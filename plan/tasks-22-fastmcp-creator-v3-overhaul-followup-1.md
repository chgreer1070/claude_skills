## Description

The `scaffold_server()` tool in `plugins/fastmcp-creator/server/server.py` generates code using
`mcp.include(FileSystemProvider(...))` (line 116) when the user requests the `filesystem-provider`
feature. However:

1. The server itself (same file, line 40) uses `mcp.add_provider(_fs_provider)` to mount its own
   FileSystemProvider.
2. The `providers.md` reference file documents `mcp.add_provider()` and `mcp.mount()` as the
   correct API — `mcp.include()` does not appear in the reference file at all.
3. If `mcp.include()` does not exist in the FastMCP v3 API, any server generated by the scaffold
   tool with `filesystem-provider` feature will fail at import or call time with an `AttributeError`.

This is an API correctness issue in the tool output — the scaffold generates broken code.

## Acceptance Criteria
- [ ] `scaffold_server()` in `server.py` generates `mcp.add_provider(FileSystemProvider(...))` for
      the `filesystem-provider` feature, not `mcp.include()`
- [ ] The generated code pattern matches the server's own usage at line 40 and the pattern
      documented in `providers.md`
- [ ] Verify against `.claude/worktrees/fastmcp/docs/servers/providers/filesystem.mdx` that
      `add_provider()` is the correct method (not `include()`)
- [ ] `uv run prek run --files plugins/fastmcp-creator/server/server.py` passes with zero errors

## Files to Modify
