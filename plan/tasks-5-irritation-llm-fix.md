### How This Currently Works: Irritation Command in session_query.py

The `irritation` command in `.claude/skills/session-historian/scripts/session_query.py` is a standalone PEP 723 script that audits Claude Code sessions for user frustration signals. When a user invokes `uv run .claude/skills/session-historian/scripts/session_query.py irritation <session_id>`, the request flows through these stages:

1. **Argument Parsing** (lines ~1106-1176, `cmd_irritation` function): Typer parses the `session_id` argument and optional `--raw` flag. The function is the entry point.

2. **Session File Loading** (lines ~1028-1051, via `_collect_correction_phrases`): The command reads the session JSONL file from `~/.claude/projects/` using `_iter_records()`. This returns a list of JSONL records (dicts with keys like `type`, `content`, `ts`).

3. **Record Filtering**: The `_collect_correction_phrases` function iterates over records and filters them:
   - Skips non-user records (checks `record.get("type") != "user"`)
   - Skips assistant tool results (checks `"toolUseResult" in record`)
   - Skips noise messages using `_is_noise()` (checks for noise prefixes like `<bash-stdout>`, `<system-reminder>`)
   - For each passing record, extracts message text via `_extract_text(record.get("content"))`

4. **Current Matching** (lines 1046-1050): The substring matcher loops through `_CORRECTION_PHRASES` (a 10-item tuple defined at lines 71-82: "stop", "don't", "revert", "wrong", etc.) and checks `if phrase in lower` where `lower = content.lower()`. On first match, appends `(timestamp, matched_phrase, content[:200])` to results and breaks.

5. **Output Accumulation** (lines ~1146-1159): Results are formatted as a list of `(timestamp, phrase, excerpt)` tuples. In `--raw` mode, prints tab-separated lines; otherwise, prints a summary count and examples using Rich formatting.

6. **Exit**: The command exits 0 on success, or returns a Typer error code on failure.

**Current Problem**: The substring matcher produces systematic false positives. Messages like "don't use tabs", "stop after step 3", and "revert to the original approach" are neutral workflow instructions, but the substring match on bare tokens like "stop" and "don't" triggers them as irritation signals. This corrupts the irritation metric, making it unreliable for session analysis. The fixed version will delegate matching to an LLM that understands context and emotional intent.

### For New Feature Implementation: What Needs to Connect

Since we're implementing LLM-based classification, the following integration points require changes:

**Import and Dependency Addition** (Task 1): The `anthropic` SDK must be added to the PEP 723 metadata block (lines 1-8) in the `dependencies` list, and `import anthropic` must be added to the top-level imports. Per CLAUDE.md PEP 723 dual-install rule, `anthropic>=0.80.0` is already a dev dependency in `pyproject.toml`, so no additional `uv add` is needed—only the inline metadata addition.

**New Function Addition** (Task 2): A new function `_classify_irritation_messages` will be added after the `_collect_correction_phrases` function (around line 1051). This function receives a list of `(timestamp, text)` tuples and makes a single batched call to `anthropic.Anthropic().messages.create()` with model `claude-haiku-4-5-20251001`. The prompt (specified in the architect spec "Prompt Design" section) classifies messages as irritation or not, and returns `(timestamp, phrase_label, excerpt)` tuples. The function does NOT use try/except around API calls or JSON parsing—errors propagate unhandled per ADR-003 (fail loud).

**Inner Loop Rewriting** (Task 3): The substring matching loop in `_collect_correction_phrases` (lines 1046-1050) is replaced with candidate accumulation. Instead of checking `if phrase in lower`, the code accumulates `candidates: list[tuple[str, str]]` of `(timestamp, content)` for each message that passes filters. After the record loop, the code calls `_classify_irritation_messages(candidates)` and assigns the result to `results`. The filter logic (type check, toolUseResult check, _is_noise check) remains unchanged.

**Constant Removal** (Task 4): The `_CORRECTION_PHRASES` constant (lines 71-82) is deleted. No other call sites exist in the file.

**Docstring Update** (Task 3): The `cmd_irritation` docstring (lines ~1115-1116) is updated to remove reference to `_CORRECTION_PHRASES` and describe LLM-based detection instead.

**Pattern Reference**: The frustration-analyzer plugin (specifically `plugins/frustration-analyzer/agents/frustration-analyst.md`) provides a reference for LLM-based emotional classification. The detection criteria in that agent's documentation directly informed the prompt design for this feature—flag escalated corrections, accusations, sarcasm aimed at the assistant; do not flag neutral instructions or self-directed frustration.

#### Component Interfaces & Signatures

**`_classify_irritation_messages` function** (new):

```python
def _classify_irritation_messages(
    messages: list[tuple[str, str]],
) -> list[tuple[str, str, str]]:
    """
    Classify user messages for genuine irritation signals using LLM judgment.

    Args:
        messages: List of (timestamp, text) tuples. Only user messages that have
                 passed noise filters are included. Empty list returns immediately
                 with no API call.

    Returns:
        List of (timestamp, phrase_label, excerpt) tuples for flagged messages.
        - timestamp: passed through unchanged from input
        - phrase_label: first 50 characters of the message text
        - excerpt: first 200 characters of the message text

    Raises:
        ValueError: if the LLM response cannot be parsed as JSON or contains
                   out-of-range message indices
        anthropic.AuthenticationError: if ANTHROPIC_API_KEY is not set (propagates unhandled)
        anthropic.APIError: any API errors propagate unhandled (no fallback)
    """
```

**`_collect_correction_phrases` function** (modified):

```python
def _collect_correction_phrases(records: list[dict]) -> list[tuple[str, str, str]]:
    """
    Collect user messages expressing correction or frustration using LLM judgment.

    Filters records by type (user only), skips toolUseResult, skips noise.
    Accumulates candidates as (timestamp, content) tuples.
    Delegates classification to _classify_irritation_messages.

    Returns:
        List of (timestamp, phrase_label, excerpt) tuples for flagged messages.
    """
```
