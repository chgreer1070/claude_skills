# Tasks: Dead Code Cleanup — Legacy Parser and Fenced YAML Recovery

All task files now use the directory format with bare YAML frontmatter. The legacy markdown
parser, fenced YAML recovery path, and their supporting infrastructure are permanently
unreachable. These tasks delete that dead code in the order that minimizes broken-import
windows. Fixes #441.

## Execution Order and Parallelization

Tasks are strictly serial. Each task removes symbols that the next task's file still imports
until that file's own deletions close the loop.

```text
T1 → T2 → T3 → T4 → T5
```

No task may start until its predecessor is COMPLETE.

### Deletion Order and Why It Is Strictly Serial

The chain T1 → T2 → T3 → T4 → T5 is ordered to minimize broken-import windows:

1. T1 removes `test_task_format_fenced_yaml.py` (which imports `detect_fenced_yaml`). Without T1, deleting `detect_fenced_yaml` in T2 causes an import error on the next `pytest` run.
2. T2 deletes `detect_fenced_yaml` from `task_format.py` AND removes its import from `implementation_manager.py` line 41 (in the same task, so pytest passes immediately after T2 without waiting for T3).
3. T3 deletes the FieldParser cluster and dead branches from `implementation_manager.py`. T2 must precede this because T3 originally was going to remove the `detect_fenced_yaml` import — T2 has already done that; T3 must verify absence, not re-attempt removal.
4. T4 inlines the field map and deletes legacy functions from `task_status_hook.py`. T3 must precede T4 so `implementation_manager.py` is already clean.
5. T5 runs all quality gates read-only with no file modifications.

### The Non-Obvious Constraint: `_legacy_field_to_yaml` Is Live

Despite its `_legacy_` prefix, `_legacy_field_to_yaml` is called from the **live** YAML branch of `add_timestamp_to_task` (line 344), not from the dead legacy branch. It converts PascalCase field names (e.g., `"LastActivity"`) to YAML snake_case keys (`"last_activity"`). Callers pass PascalCase names; this function is the only place that conversion happens. Deleting it without inlining its mapping first would silently corrupt all timestamp writes. T4's first action is to read the function body verbatim from the file and verify the four-entry map matches the spec before editing anything.
