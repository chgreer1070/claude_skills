# Product

## Register

product

## Users

One developer (Jamie) making architectural decisions about the beads backend integration for the Development Harness Claude Code plugin. Opens the page in a browser tab alongside a code editor. Skims tables, compares fidelity, finds a specific row — then closes the tab. Not shared externally.

## Product Purpose

Technical reference document comparing beads vs GitHub as a backend for the DH plugin: fidelity mapping tables, architecture diagrams, decision forks, gap analysis, implementation checklist, CLI specification. Success = finding the answer to "does beads support X?" in under 10 seconds.

## Brand Personality

Precise, confident, editorial. A printed technical white paper, not a SaaS admin panel. Authoritative without being corporate.

## Anti-references

- GitHub Flavored Markdown rendered as HTML ("printed README" look — no visual ambition)
- Notion / Confluence wiki (emoji bullets, pastel callout card grids, generic SaaS wiki feel)

## Design Principles

1. Information first: every visual decision must aid comprehension, not decoration
2. Color earns its place: color signals meaning (section type, fidelity, decision criticality) — never ornament
3. Weight over opacity: hierarchy through type weight variation, not grayed-out text
4. Confidence at every scale: headlines feel editorial; tables feel authoritative
5. Density without clutter: information packs tightly, breathing room between sections is generous

## Accessibility & Inclusion

WCAG AA minimum. Light mode only. Standard browser zoom. No motion dependencies.
