# Quick Start: Research Integration Opportunities

This guide shows you how to process research files to add Integration Opportunities sections.

## Quick Process (Single File)

In Claude Code, use the Task tool:

```
Process research file: research/developer-tools/loguru.md
```

That's it! The research-context-agent will:
1. Read and understand the file
2. Search for connections across the codebase
3. Append/update the Integration Opportunities section

## Batch Processing All Files

### Option 1: Interactive Script (Recommended)

```bash
./batch-process-research.sh
```

This will:
- Show you each file to process
- Display the Task tool prompt to use
- Track your progress
- Let you skip files that already have the new format

### Option 2: Process by Category

```bash
./batch-process-research.sh --category developer-tools
./batch-process-research.sh --category mcp-ecosystem
```

### Option 3: Manual Task Tool

For each file you want to process, use this Task tool prompt:

```
You are the research-context-agent as defined in /absolute/path/to/.claude/agents/research-context-agent.md

Please process this research file:
/absolute/path/to/research/category/filename.md

Follow the three-phase process:
1. Absorb: Read and understand the research file
2. Search & Match: Find connections across the 5 dimensions
3. Append: Add or REPLACE the Integration Opportunities section

Use the structured format with:
- "### Enhances Existing" table
- "### New Skill Candidates" list
- "### New MCP Server Candidates" list  
- "### Cross-References" list

Remember: Concrete over vague, skip empty sections, no false positives.
```

## What Gets Added

Each processed file gets a section like this:

```markdown
---

## Integration Opportunities

> Auto-generated by research-context-agent. Review before acting.

### Enhances Existing

| Target | Type | How |

| `plugins/holistic-linting/skills/holistic-linting/` | skill | Add jscpd duplicate detection as a pre-lint step before running other linters, using the --threshold and --mode json flags to integrate with existing lint report formats. |

### New Skill Candidates

- **duplication-refactoring**: Teach Claude to detect code duplication using jscpd, analyze the duplicate blocks, propose DRY refactoring strategies, and apply extract-method or extract-class patterns to eliminate duplication while preserving behavior.

### Cross-References

- Related research: `research/developer-tools/repomix.md` — Repomix provides whole-codebase AI analysis context; jscpd results could annotate repomix output to flag duplicate patterns before AI analysis.
```

## Verifying Results

After processing a file, check:

1. **New section exists**: Search for "## Integration Opportunities"
2. **Has new format**: Look for "### Enhances Existing" table
3. **Descriptions are concrete**: Not vague like "could be useful"
4. **Empty sections omitted**: If no MCPs found, no MCP section
5. **Existing content preserved**: Everything above section is unchanged

## Troubleshooting

**Q: Agent produces vague suggestions**
A: Emphasize "concrete over vague" in your prompt. Give examples.

**Q: Too many false positives**
A: Remind agent: "Only genuine, specific connections"

**Q: Missing obvious connections**  
A: List the specific skills/agents you expect it to find

**Q: Wrong format**
A: Verify you're referencing the agent at `.claude/agents/research-context-agent.md`

## After Processing

1. Review the added sections for accuracy
2. Verify cross-references make sense
3. Check that connections are actionable
4. Commit changes: `git commit -m "feat: add integration opportunities to [category]"`
