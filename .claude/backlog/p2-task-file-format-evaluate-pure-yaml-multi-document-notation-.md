## Groomed (2026-03-02)

### Priority

7/10 — The fenced-code-block bug in swarm-task-planner causes null agents and NOT_STARTED defaults on every generated plan, requiring manual correction. Script-only writes enforcement prevents a parallel class of silent corruption. Both are correctness issues in the task file pipeline.

### Impact

- Blocks: Any SAM plan generated by swarm-task-planner where the planner uses the documented template — tasks produce agent: null, status: not-started silently when the parser cannot read the fenced YAML block
- Bottleneck: /implement-feature queries ready-tasks from implementation_manager.py; null agents cause incorrect delegation or no delegation

### Scope

Two targeted changes, scoped separately:

1. Agent template fix: Remove the fenced yaml wrapper from the task structure template in both swarm-task-planner agent files so the model generates raw frontmatter
2. Format validator: Add a validation function (or extend parse_task_content) that raises an actionable error when has_yaml_frontmatter() returns False and the file appears to contain a fenced YAML block — instead of silently falling through to legacy parsing
3. Script-only writes policy: Document in TASK_FILE_FORMAT.md that task data files are written only by the four designated scripts; add a check in parse_task_content that warns when called on content that looks like manually written markdown bold-fields format

Note: Pure YAML multi-document notation evaluation is a separate design research task; the immediate correctness fix does not require it.

### Output / Evidence

The fenced code block issue is reproduced by examining the swarm-task-planner template at:
- plugins/python3-development/agents/swarm-task-planner.md lines 249-265
- plugins/development-harness/agents/swarm-task-planner.md lines 248-264

Both templates show the task frontmatter wrapped in a fenced yaml code block. The parser at task_format.py line 78 requires content.startswith("---
") — a fenced block fails this check and falls through to legacy markdown parsing, which finds no ## Task header, returning an empty list.
